Создать iOS приложение, интерфейс которого содержит таббар и 2
таблицы, каждая вкладка таббара соответствует своей таблице.
Таблицы должны являться визуальным представлением данных
модели Person (uint personId, string personName, string personCountry).
Первая таблица содержит имя человека и его порядковый номер (uint
personId, string personName).
Вторая таблица содержит имя человека и страну его проживания
(string personName, string personCountry).
Обе таблицы имеют общий датасорс, а также должно быть несколько
ячеек, отображающих одного и того же человека.
Модельный объект Person в данном случае обладает свойствами
синглтона. Уникальность модели обеспечивает поле personId.
В отдельном потоке/очереди, который запускается при
переключении свитчера, должна запускаться циклическая процедура
автоматической выборки случайного пользователя и изменение его
данных. При изменении данных происходит обновление всех
соответствующих элементов интерфейса для данной модели. При этом
главный поток не должен блокироваться на заметное для пользователя
время.
Приложение должно стабильно работать и обеспечивать гладкую
прокрутку таблицы.
Дизайн программы, способ хранения моделей данных, как и все
неоднозначные нюансы задания, которые не были конкретизированы
выше, решаются на усмотрение соискателя.
